# Security Policy

StringWiggler is a small, deliberately-pointless hobby toy, but if you spot a
genuine security problem it would still be good to hear about it.

## Supported Versions

| Version | Supported         |
| ------- | ----------------- |
| 0.x.x   | Yes (development) |

Everything is currently pre-release (0.x) and under active development. Once a
v1.0 exists, this table will be updated.

## Reporting a Vulnerability

**Please do not report security issues through public GitHub issues.**

Instead, use either of the following:

1. **Preferred:** [GitHub Security Advisories](https://github.com/MatejGomboc/StringWiggler/security/advisories/new)
   to report privately.
2. **Alternative:** email the repository owner at <matejg04@gmail.com>.

### What to Include

- A clear description of the problem
- Steps to reproduce
- The potential impact, as you see it
- Any suggested fix (optional, but appreciated)

### What to Expect

This is a one-person hobby project, so reports are handled on a best-effort
basis with **no guaranteed response timeline**. That said, reports will be read
and, where reasonable, acted upon. If you would like credit in any eventual fix,
say so — otherwise reports are treated as confidential. Please allow reasonable
time to address an issue before any public disclosure.
